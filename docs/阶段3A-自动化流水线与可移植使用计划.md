# Stage 3A：项目产品化与自动化流水线 MVP

## 1. 目标

将项目从"本地试点产物"变成"clone 后可运行的 PDF→study-kb 框架"。

**核心交付**：
- `pipeline.py`：统一 CLI 入口，支持 status/validate/coverage/publish/make-tasks
- `CLAUDE_CODE_WORKFLOW.md`：用户操作手册
- `requirements.txt`：依赖声明
- 测试：验证流水线各命令的正确性

---

## 2. 当前架构回顾

### 2.1 讲义生成流程（Authoring）

**工具**：Claude Code + `section-lesson-authoring` skill

**输入**：
- 原始 PDF（`books/{book-id}/input/*.pdf`）
- section-manifest.yaml 中的 section 元数据
- source-slice.md（从 PDF 提取的原文切片）

**输出**：
- section-lesson-draft.md（草稿，frontmatter 中 review_status: draft, generation_stage: draft）

**流程**：
1. 用户在 Claude Code 中执行 `make-tasks` 生成任务包
2. Claude Code 根据任务包调用 `section-lesson-authoring` skill
3. Skill 读取 source-slice.md，按模板生成讲义草稿
4. 草稿写入 `pipeline-workspace/staging/{section-id}/section-lesson-draft.md`

### 2.2 讲义审校流程（Review）

**工具**：Claude Code + `section-lesson-review` skill

**输入**：
- section-lesson-draft.md
- source-slice.md

**输出**：
- review-decision.yaml（审校决策）
- review-report.md（审校报告）

**流程**：
1. 用户在 Claude Code 中执行 `make-tasks` 生成任务包
2. Claude Code 根据任务包调用 `section-lesson-review` skill
3. Skill 对比草稿与原文，检查忠实性、可学习性、结构完整性
4. 输出审校决策（accept/revise/reject）和报告

### 2.3 结构校验（Validate）

**工具**：`scripts/validate_section_lesson.py`

**能力**：
- 检查 frontmatter 存在且可解析
- 检查必填字段（id, type, source_title, book_order, importance, difficulty, formula_risk, review_status, generation_stage）
- 检查字段值合法性
- 检查必备章节标题（12 个）

**局限**：
- 不能检查内容忠实性
- 不能检查公式正确性
- 不能检查学习价值
- 只做结构校验，不做语义校验

### 2.4 Codex 审核

**定位**：可选的质量增强步骤，不是最终用户必需步骤。

**作用**：
- 提供独立的代码审阅视角
- 发现潜在的设计问题
- 验证统计数字的一致性

**说明**：
- 用户可以跳过 Codex 审核，直接使用 pipeline 完成工作
- Codex 审核主要用于开发阶段的质量保证
- 生产环境中，用户只需依赖 validate 和人工抽样

## 3. pipeline.py 命令设计

### 3.1 status

显示当前项目状态。

**输出**：
- 总小节数
- 各状态小节数（registered/drafted/reviewed/published）
- 已发布小节列表

**示例**：
```bash
python scripts/pipeline.py status --book 博弈论白皮书
```

### 3.2 validate

校验指定小节的讲义文件。

**输入**：section-id 或 --all

**输出**：
- 通过/失败状态
- 错误列表

**示例**：
```bash
python scripts/pipeline.py validate --book 博弈论白皮书 --section GTW-002-01
python scripts/pipeline.py validate --book 博弈论白皮书 --all
```

### 3.3 coverage

显示覆盖报告。

**输出**：
- 总小节数
- 已发布数
- 覆盖率
- 按章节统计

**示例**：
```bash
python scripts/pipeline.py coverage --book 博弈论白皮书
```

### 3.4 publish

发布指定小节到 study-kb。

**输入**：section-id 或 --all-reviewed

**前置检查**：
- section-manifest.yaml 中 status=reviewed
- review-decision.yaml 存在且 decision=accept
- section-lesson-draft.md 存在

**输出**：
- 发布结果
- 更新后的 manifest 状态

**示例**：
```bash
python scripts/pipeline.py publish --book 博弈论白皮书 --section GTW-002-01
python scripts/pipeline.py publish --book 博弈论白皮书 --all-reviewed
```

### 3.5 make-tasks

生成 Claude Code 可执行的任务包。

**输入**：section-id 或 --all-registered

**输出**：
- author 任务包（JSON）
- review 任务包（JSON）

**任务包格式**：
```json
{
  "task_type": "author",
  "section_id": "GTW-001-01",
  "book_id": "博弈论白皮书",
  "inputs": {
    "source_slice": "books/博弈论白皮书/pipeline-workspace/staging/GTW-001-01/source-slice.md",
    "manifest": "books/博弈论白皮书/config/section-manifest.yaml",
    "template": "templates/section-lesson.template.md"
  },
  "outputs": {
    "draft": "books/博弈论白皮书/pipeline-workspace/staging/GTW-001-01/section-lesson-draft.md"
  },
  "success_criteria": [
    "frontmatter 包含所有必填字段",
    "包含 12 个必备章节标题",
    "通过 validate_section_lesson.py 校验"
  ],
  "skill": "section-lesson-authoring"
}
```

## 4. 用户工作流

### 4.1 Clone 后首次使用

```bash
# 1. 克隆项目
git clone <repo-url>
cd pdf-to-study-kb

# 2. 安装依赖
pip install -r requirements.txt

# 3. 查看项目状态
python scripts/pipeline.py status --book 博弈论白皮书

# 4. 查看覆盖报告
python scripts/pipeline.py coverage --book 博弈论白皮书
```

### 4.2 生成新小节

```bash
# 1. 生成任务包
python scripts/pipeline.py make-tasks --book 博弈论白皮书 --section GTW-001-01

# 2. 在 Claude Code 中执行 author 任务
# Claude Code 会自动调用 section-lesson-authoring skill

# 3. 在 Claude Code 中执行 review 任务
# Claude Code 会自动调用 section-lesson-review skill

# 4. 校验生成的讲义
python scripts/pipeline.py validate --book 博弈论白皮书 --section GTW-001-01

# 5. 发布讲义
python scripts/pipeline.py publish --book 博弈论白皮书 --section GTW-001-01
```

### 4.3 批量处理

```bash
# 1. 为所有未发布小节生成任务包
python scripts/pipeline.py make-tasks --book 博弈论白皮书 --all-registered

# 2. 在 Claude Code 中批量执行任务
# Claude Code 会按任务包顺序执行

# 3. 校验所有讲义
python scripts/pipeline.py validate --book 博弈论白皮书 --all

# 4. 发布所有已审校讲义
python scripts/pipeline.py publish --book 博弈论白皮书 --all-reviewed
```

## 5. 人工介入点

### 5.1 必须人工处理

- **高公式风险小节**（formula_risk: high）：必须人工验证公式正确性
- **审校失败小节**：review-decision.yaml 中 decision=revise 或 reject
- **校验失败小节**：validate 返回 passed=false

### 5.2 建议人工抽样

- **随机抽样**：每 10 个小节随机抽 1 个进行人工审阅
- **关键章节**：第一章和最后一章建议全部人工审阅
- **新类型**：首次处理的新类型小节（如新的 formula_risk 级别）

### 5.3 可选人工参与

- **Codex 审核**：开发阶段使用，生产环境可选
- **概念桥接**：Knowledge-Bridges 生成，可选
- **阅读地图调整**：MOC 生成后的微调
