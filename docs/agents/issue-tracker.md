# Issue Tracker

This repository tracks work in GitHub Issues.

Repository:

- GitHub repo: `Iabstergo1/pdf-to-study-kb`
- Remote URL: `https://github.com/Iabstergo1/pdf-to-study-kb.git`

Engineering skills should use GitHub Issues as the source of truth for planned work, bug reports, migration tasks, and follow-up implementation slices.

When creating issues for the semantic-unit migration, prefer small phase-aligned issues rather than one large tracking issue. Each issue should include:

- Scope
- Files likely affected
- Acceptance criteria
- Verification commands
- Whether the work touches legacy section flow, semantic unit flow, or both

For local planning before GitHub issues exist, use conversation summaries or temporary markdown notes only as drafts. Do not treat local scratch notes as the canonical tracker unless this document is changed later.
