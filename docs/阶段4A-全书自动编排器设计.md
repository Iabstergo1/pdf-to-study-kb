# 阶段 4A：Claude Code 全书编排器设计

## 目标

项目只保留 Claude Code 模式。用户把 PDF 放入本地后，Python 流水线负责确定性工作，Claude Code 负责 author/review 内容生成，最终产出可直接用 Obsidian 打开的本地知识库。

目标体验：

```text
PDF -> init-book -> inventory -> extract -> make-tasks
    -> Claude Code author/review
    -> mark-reviewed -> publish -> study-kb
```

## 非目标

- 不在项目代码中保存密钥或账号 token。
- 不提供任何独立模型执行器；内容生成只通过 Claude Code 任务队列完成。
- 不把原始 PDF、个人配置、运行日志或生成知识库提交到公开仓库。

## run-book 命令

```powershell
python scripts/pipeline.py run-book --book <book-id> --executor claude-code-queue
python scripts/pipeline.py run-book --book <book-id> --executor claude-code-queue --dry-run
python scripts/pipeline.py run-book --book <book-id> --executor claude-code-queue --section <section-id>
```

职责：

- `--dry-run`：只输出计划，不创建 runs，不写 Obsidian 文件。
- 正常运行：生成或刷新 Claude Code 任务包，创建 run-state，更新 Obsidian Home/MOC/QA 文件。
- `--section`：用于单小节调试。
- `--resume`：读取最近一次 run-state，展示断点状态。

## 状态机

全局状态：

```text
books/<book-id>/pipeline-workspace/runs/<run-id>/run-state.yaml
```

小节状态：

```text
books/<book-id>/pipeline-workspace/runs/<run-id>/sections/<section-id>.yaml
```

核心状态：

- `not_started`
- `authoring`
- `reviewing`
- `reviewed`
- `published`
- `failed`
- `needs_human_review`

## Claude Code 任务队列

任务包目录：

```text
books/<book-id>/pipeline-workspace/tasks/
```

每个小节两个任务：

- `<section-id>_author.json`
- `<section-id>_review.json`

Claude Code 必须按顺序执行：

1. 读取 author 任务包。
2. 使用 `.claude/skills/section-lesson-authoring/SKILL.md` 生成 draft。
3. 读取 review 任务包。
4. 使用 `.claude/skills/section-lesson-review/SKILL.md` 生成 review 报告和决策。
5. 不直接写 `study-kb/`。

## 门禁

`mark-reviewed` 只接受同时满足以下条件的小节：

- `review-decision.yaml` 存在。
- `decision: accept`。
- `required_fixes: []`。
- `section-lesson-draft.md` 存在。
- draft 通过 `validate_section_lesson.py`。

`publish` 只发布 `status: reviewed` 的小节，并把发布后的 Markdown 写入：

```text
books/<book-id>/study-kb/Section-Lessons/
```

## Obsidian 输出

`run-book` 在非 dry-run 模式下更新：

- `Home.md`
- `Learning-Maps/MOC-全书学习地图.md`
- `Learning-Maps/MOC-入门最短路线.md`
- `Learning-Maps/MOC-难点与推导重点路线.md`
- `Source-QA/小节覆盖报告.md`
- `Source-QA/高风险内容清单.md`

这些文件只根据 manifest 和已发布讲义生成，不调用模型。

## 公开仓库边界

提交：

- Python 流水线代码
- tests
- templates
- schemas
- docs
- Claude Code skills
- `.env.example`
- `books/.gitkeep`

不提交：

- `.env`
- 原始 PDF
- `books/*/pipeline-workspace/`
- `books/*/study-kb/`
- `books/*/config/personal-context.yaml`
