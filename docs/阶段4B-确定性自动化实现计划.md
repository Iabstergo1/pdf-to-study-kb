# 阶段 4B：Claude Code 队列自动化实现计划

## 范围

本阶段只实现 Claude Code 队列模式：

- `scripts/run_state.py`：记录 run-state 和 section-state。
- `scripts/obsidian_output.py`：生成 Obsidian Home、MOC 和 QA 文件。
- `scripts/run_book.py`：注册全书级编排入口。
- `scripts/pipeline.py`：暴露 `run-book --executor claude-code-queue`。
- `tests/test_manual.py`：覆盖 dry-run、任务队列、Obsidian 输出和真实书籍只读 smoke。

明确不做：

- 不实现独立模型执行器。
- 不保存或读取模型密钥。
- 不把示例 PDF 或生成知识库提交到公开仓库。
- 不绕过 Claude Code skills 直接生成学习讲义。

## run_state.py

公开方法：

- `create_run(book_id, config, sections)`
- `load_latest_run(book_root)`
- `load_run(run_id, book_root=None)`
- `update_stage(run_state, stage_name, status, **kwargs)`
- `update_section(run_state, section_id, action, result, **kwargs)`
- `get_next_sections(run_state, batch_size)`
- `calculate_progress(run_state)`
- `finalize(run_state, status="completed")`
- `save(run_state)`

输出位置：

```text
books/<book-id>/pipeline-workspace/runs/<run-id>/
├── run-state.yaml
└── sections/
    └── <section-id>.yaml
```

数据来源：

- `config/section-manifest.yaml`
- `study-kb/Section-Lessons/*.md`

## run_book.py

参数：

```text
--book <book-id>
--executor claude-code-queue
--publish accepted-only|manual
--section <section-id>
--resume
--dry-run
--batch-size <n>
--max-revision-retry <n>
```

行为：

- `--dry-run`：只打印批次计划和风险统计，不写文件。
- 正常运行：创建 run-state，调用 `make-tasks` 生成 Claude Code 任务包，更新 Obsidian 输出。
- `--section`：只为指定小节生成任务包。
- `--resume`：读取最近 run-state 并打印进度。

## pipeline.py

注册命令：

```powershell
python scripts/pipeline.py run-book --book my-book --executor claude-code-queue
```

CLI 不暴露任何模型接口相关参数。

## 测试计划

默认 pytest 门禁：

- run-state 创建、加载和进度统计。
- Obsidian 输出生成。
- run-book dry-run 零写入。
- run-book 生成 Claude Code 任务包和 Obsidian 输出。
- run-book 拒绝非 Claude Code executor。
- 既有 init-book / inventory / extract / make-tasks / mark-reviewed / publish 测试不回退。

真实书籍只读 smoke：

- 示例书籍存在时检查 status、coverage、validate、manifest 零写入。
- 示例书籍不存在时跳过，不影响公开仓库测试。
