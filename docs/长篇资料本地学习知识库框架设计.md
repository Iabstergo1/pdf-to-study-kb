# 长篇资料本地学习知识库框架设计

## 1. 文档目的

本文档定义一个可复用框架，用于将长篇 PDF、教材、白皮书、研究入门资料或课程讲义编译为本地学习知识库。

该框架优先解决以下问题：

1. 长篇电子文档线性阅读效率低，难以按学习目标筛选重点。
2. 普通摘要容易丢失原文章节脉络、来源页码、推导上下文和学习难点。
3. 仅靠一次性大模型长任务，容易出现中断、风格漂移、漏节、误解和无法复跑。
4. 只做碎片化原子笔记，容易把教材或长文档的学习主线切碎。
5. 个人知识沉淀不能停留在“复述原文”，还应识别哪些内容值得进入长期知识体系、项目、写作和后续问题清单。

本文档当前以 `博弈论研究完全自学入门-自救白皮书.pdf` 为首个试点资料，但框架本身不得绑定于单一论文、单一学科或单一 Obsidian 插件生态。

## 2. 核心设计决策

### 2.1 采用 Markdown-first，而不是 Obsidian-first

框架的核心资产是本地 Markdown 文件、结构化元数据、来源映射、审校状态和自动化流水线。

Obsidian 是主要阅读界面之一，知识库应兼容 Obsidian 的双链、MOC、标签、Frontmatter、搜索和图谱使用习惯，但质量控制、任务恢复、审校门禁和文件契约不应依赖某个 Obsidian 插件。

### 2.2 最终主阅读体是小节学习讲义

最终给人主要阅读的不是解析日志、原始抽取文本、manifest 或零散概念卡片，而是按原书小节生成的高密度学习讲义。

每篇小节学习讲义应帮助读者完成：

1. 快速定位本节要解决的问题。
2. 区分必须掌握、首遍可略读和需要回原文精读的内容。
3. 理解核心概念、直觉、模型结构或推导骨架。
4. 在需要时准确回到原文页码、图表、例题和完整推导。

### 2.3 保留书的结构，再叠加多路径阅读

知识库不得把一本长文档直接打散成孤立碎片。先保留原始章节和小节主线，再叠加：

- 阅读地图。
- 概念复习页。
- 术语、符号和例题索引。
- 个人知识桥接页。
- 来源和质量追踪页。

### 2.4 自动化采用 manifest 驱动

长任务状态必须写入本地 manifest，不能依赖单个长会话的上下文记忆。

每个小节都应具有明确状态，例如：

```text
registered
-> extracted
-> drafted
-> reviewed
-> accepted
-> published
```

需要返工或人工复核的状态也应显式记录，例如：

```text
needs_reextract
needs_formula_review
needs_human_review
rejected
```

### 2.5 多 Agent 并行必须建立在写入边界上

并行的基本单位是已切分的小节任务。Worker 不应自由修改全局索引、总术语表、最终知识库首页或共享概念主文件。

多个 agent 可以并行生成不同小节的草稿和审校报告，最终由 integrator 只合并通过门禁的产物。

### 2.6 个人知识桥接层必须通用化

个人知识桥接层用于把来源知识接入个人知识体系，而不是把框架写死为“与某一章论文连接”。

它应回答：

1. 这份资料中哪些内容值得进入长期知识库。
2. 哪些内容只服务于原文上下文，不需要额外沉淀。
3. 哪些概念、方法、判断可以连接到既有主题。
4. 哪些内容可进入项目、写作、研究、决策或后续阅读。
5. 哪些问题尚未真正理解，应保留为问题而不是伪装成已掌握结论。

## 3. 目标与非目标

### 3.1 目标

第一阶段框架目标如下：

1. 将长篇资料转换为本地、可迁移、可审校的 Markdown 学习知识库。
2. 使知识库可直接用 Obsidian 打开并高效阅读。
3. 支持按原书顺序、学习目标、难点专题和个人上下文多路径阅读。
4. 为每个小节保留来源定位、轻重分级、学习难度和回原文条件。
5. 利用 Claude Code 或 Agent SDK 的 agent、skill、hook 和自动化编排能力处理大规模分节任务。
6. 让自动化任务可中断、可恢复、可局部重跑、可审计。
7. 为后续导出 book skill、plugin 或接入个人 Vault 保留扩展空间。

### 3.2 非目标

第一阶段不追求：

1. 用知识库彻底替代原始 PDF。
2. 让 AI 全自动保证学术正确。
3. 一开始就搭建复杂 RAG、向量库或知识图数据库。
4. 直接自动修改真实个人 Obsidian Vault。
5. 为每一份资料自动生成大量弱关联双链。
6. 首版就覆盖所有文档格式、所有扫描质量和所有复杂版面。
7. 将个人第二大脑体系、Inbox 清理、日报、周报和长篇资料编译揉成一个过大的系统。

## 4. 从上到下的知识库结构

### 4.1 总体层次

```text
原始长篇资料
  ↓
结构化学习知识库
  ├── 导航层
  │   └── 告诉读者先读什么、按什么路线读
  ├── 小节学习层
  │   └── 承担逐节学习的主阅读体
  ├── 概念复习层
  │   └── 抽取跨小节可复用的概念与方法
  ├── 个人知识桥接层
  │   └── 接入个人主题、项目、写作、问题与后续阅读
  └── 溯源与质量层
      └── 保证可回原文、可审校、可复跑
```

### 4.2 最终给人阅读的内容

| 阅读对象 | 角色 |
| --- | --- |
| `Home` 与 `Learning-Maps` | 主入口，决定阅读路线 |
| `Section-Lessons` | 主阅读体，承担逐小节学习 |
| `Concept-Cards` 与 `Glossary` | 辅助理解、查阅和复习 |
| `Knowledge-Bridges` | 个人沉淀、项目连接、写作线索和问题保留 |
| `Source-QA` 与原 PDF | 回原文、看风险、核对来源 |

知识库的理想使用方式是：

```text
先看首页和阅读地图
-> 选择一条阅读路线
-> 阅读小节学习讲义
-> 遇到概念跳转概念页
-> 遇到推导、图表、歧义和例题回原文
-> 读完后查看个人知识桥接页
```

### 4.3 与原文的关系

知识库服务于学习效率和知识定位，原始资料仍是权威源。

| 场景 | 优先阅读知识库 | 应回原文 |
| --- | --- | --- |
| 快速理解章节主旨 | 是 | 通常不用 |
| 按目标筛选重点 | 是 | 通常不用 |
| 概念复习 | 是 | 通常不用 |
| 完整例题 | 先看知识库 | 常需要 |
| 完整公式推导 | 先看骨架 | 高概率需要 |
| 核对作者原意 | 不足 | 必须 |
| 正式引用 | 不足 | 必须 |

## 5. 最终学习知识库目录

建议每份资料生成一个独立的 `study-kb` 目录。首版结构如下：

```text
study-kb/
├── Home.md
├── Learning-Maps/
│   ├── MOC-全书学习地图.md
│   ├── MOC-入门最短路线.md
│   ├── MOC-难点与推导重点路线.md
│   └── MOC-个人上下文优先路线.md
├── Section-Lessons/
│   ├── 小节学习讲义-001.md
│   ├── 小节学习讲义-002.md
│   └── ...
├── Concept-Cards/
│   ├── 概念卡片-示例.md
│   └── ...
├── Knowledge-Bridges/
│   ├── 本资料与我的知识库连接总览.md
│   ├── 值得沉淀的长期知识.md
│   ├── 可进入项目或写作的材料.md
│   ├── 值得保留的问题.md
│   └── 后续阅读线索.md
├── Glossary/
│   ├── 术语表.md
│   ├── 符号表.md
│   └── 例题与案例索引.md
├── Source-QA/
│   ├── 原文页码索引.md
│   ├── 小节覆盖报告.md
│   ├── 高风险内容清单.md
│   └── 未解决问题.md
└── Assets/
```

### 5.1 主入口

`Home.md` 应回答：

1. 这份资料是什么。
2. 这份知识库如何阅读。
3. 当前有哪些阅读路线。
4. 哪些内容是高优先级。
5. 哪些内容尚未审校或需要回原文。

### 5.2 阅读地图

阅读地图是导航，不是摘要。至少支持：

1. 按原文顺序阅读。
2. 入门最短路线。
3. 难点或推导重点路线。
4. 个人上下文优先路线。

### 5.3 小节学习讲义

小节学习讲义保留长文档主线，是整个知识库的核心学习单元。

每篇讲义应在学习上独立可读，但不得假装完全替代原文。

### 5.4 概念卡片

概念卡片只承载跨小节稳定复用的概念、方法、判断框架和术语辨析。

概念卡片不应先于小节学习讲义泛滥生成。首版应从已审校小节中抽取候选概念，再由 synthesis 阶段合并。

### 5.5 个人知识桥接页

个人知识桥接页不等于“自动写入个人 Vault”。

首版应生成桥接建议和桥接页，帮助用户决定：

1. 哪些内容值得长期沉淀。
2. 哪些内容值得连接到既有主题或项目。
3. 哪些内容可成为写作、研究、方案和决策素材。
4. 哪些疑问应继续保留。

## 6. 阅读方式与增删改查

### 6.1 阅读

知识库支持五种阅读方式：

| 阅读方式 | 入口 | 目的 |
| --- | --- | --- |
| 原文顺序学习 | 全书学习地图 | 系统自学 |
| 目标导向学习 | 个人上下文优先路线 | 先学当前最有价值内容 |
| 难点专题学习 | 推导重点路线、概念卡片 | 处理高难度知识 |
| 快速查阅 | 术语表、符号表、搜索 | 忘记定义时快速定位 |
| 回原文精读 | 页码索引与回原文条件 | 看完整推导、图表和例题 |

### 6.2 增

新增内容分为三类：

| 内容类型 | 建议去向 |
| --- | --- |
| 来源资料的新小节 | `Section-Lessons` |
| 跨节可复用知识 | `Concept-Cards` |
| 个人理解、项目连接和问题 | `Knowledge-Bridges` |

### 6.3 删

最终知识库不应由自动化流水线直接大规模删除内容。

删除前应检查：

1. 是否被阅读地图引用。
2. 是否存在入链。
3. 是否被个人知识桥接页引用。
4. 是否应先标记为 `deprecated` 或 `merged`。

### 6.4 改

修改时必须区分内容来源：

| 内容区域 | 自动化重跑策略 |
| --- | --- |
| AI 生成摘要与学习讲义草稿 | 可重跑 |
| 审校结论 | 通过 review 流程更新 |
| 用户个人理解 | 默认不可覆盖 |
| 个人桥接结论 | 默认建议式更新 |

### 6.5 查

查询能力分三层：

1. Obsidian 的全文搜索、双链、标签、MOC 和图谱。
2. Frontmatter 与 manifest 的结构化查询。
3. Agent 基于已审校资料的问答和检索。

Agent 查询应优先基于已 `reviewed` 或已 `published` 的知识单元，并返回来源小节和原文定位。

## 7. 小节学习讲义规格

### 7.1 Frontmatter

建议每篇小节学习讲义至少包含：

```yaml
---
id: SAMPLE-001-01
type: section-lesson
source_title: 来源资料标题
source_locator:
  pages: [1, 4]
book_order: "1.1"
importance: A
difficulty: 3
personal_kb_relevance: high
formula_risk: low
review_status: reviewed
generation_stage: published
---
```

### 7.2 正文结构

```markdown
# 小节标题

## 学习定位
## 先记住的结论
## 必须掌握
## 首遍可略读
## 核心概念
## 模型结构、论证骨架或推导骨架
## 直觉解释
## 容易误解的点
## 与个人知识体系的连接候选
## 自测问题
## 何时回原文
## 原文定位
```

### 7.3 内容分层

每篇讲义应明确区分：

1. 来源忠实压缩。
2. 为学习而做的解释补充。
3. 个人知识桥接候选。

不应把作者原意、模型补充解释和用户个人迁移判断混写成同一种口吻。

## 8. 自动化工程结构

建议将通用自动化工程和单份资料实例分开。

```text
pdf-to-study-kb/
├── README.md
├── CLAUDE.md
├── .claude/
│   ├── agents/
│   ├── skills/
│   ├── hooks/
│   └── settings.json
├── schemas/
├── templates/
├── scripts/
├── docs/
└── books/
    └── sample-book/
        ├── input/
        ├── config/
        ├── pipeline-workspace/
        └── study-kb/
```

### 8.1 通用引擎层

通用引擎层应承载：

1. agent 角色定义。
2. skills。
3. hooks。
4. schema。
5. 模板。
6. 校验脚本。
7. 导出器。

### 8.2 资料实例层

每份资料至少包含：

```text
books/<book-id>/
├── input/
├── config/
│   ├── book-profile.yaml
│   ├── study-profile.yaml
│   ├── personal-context.yaml
│   └── section-manifest.yaml
├── pipeline-workspace/
└── study-kb/
```

### 8.3 工作区与阅读区分离

`pipeline-workspace` 只存放：

- 原始抽取。
- 中间草稿。
- review 结果。
- 任务日志。
- 质量报告。

`study-kb` 只存放发布后的阅读产物。

## 9. 配置契约

### 9.1 `book-profile.yaml`

描述资料本身：

```yaml
book_id: sample-book
title: 来源资料标题
source_type: pdf
language: zh
domain: general
source_authority: original-source
expected_structure:
  unit: section
  preserve_source_order: true
risk_flags:
  formulas: unknown
  tables: unknown
  scanned_pages: unknown
```

### 9.2 `study-profile.yaml`

描述学习输出风格：

```yaml
lesson_style:
  primary_unit: section-lesson
  density: high
  preserve_context: true
  allow_concept_cards: true
importance_levels:
  - A
  - B
  - C
reading_routes:
  - source-order
  - beginner-fast-path
  - difficult-topics
  - personal-context-priority
must_include:
  - learning-position
  - core-takeaways
  - must-master
  - first-pass-skippable
  - intuitive-explanation
  - misunderstandings
  - source-return-conditions
```

### 9.3 `personal-context.yaml`

描述个人知识桥接策略：

```yaml
bridge_policy:
  generate_bridge_notes: true
  do_not_modify_existing_personal_kb: true
  output_mode: suggestions-first
contexts:
  - name: 当前关注主题
    type: knowledge-area
    focus:
      - 示例主题
bridge_targets:
  - concepts-worth-keeping
  - project-relevance
  - writing-material
  - open-questions
  - follow-up-reading
```

### 9.4 `section-manifest.yaml`

描述小节任务：

```yaml
sections:
  - id: SAMPLE-001-01
    source_order: "1.1"
    title: 小节标题
    source_locator:
      pages: [1, 4]
    status: registered
    extraction_risk: unknown
    formula_risk: unknown
    publish_status: not-published
```

## 10. 多 Agent 流水线

### 10.1 阶段

```text
Stage 0 资料登记
  ↓
Stage 1 解析与来源映射
  ↓
Stage 2 章节和小节切分
  ↓
Stage 3 小节学习讲义并行生成
  ↓
Stage 4 小节审校
  ↓
Stage 5 概念页和个人知识桥接页生成
  ↓
Stage 6 阅读地图与索引合并
  ↓
Stage 7 最终知识库校验与导出
```

### 10.2 Agent 角色

| Agent | 主要职责 | 不应做什么 |
| --- | --- | --- |
| `orchestrator` | 读配置、推进阶段、派发任务 | 不写最终学习内容 |
| `pdf-ingest-agent` | 解析来源、生成页码与质量报告 | 不总结知识 |
| `section-planner` | 生成小节任务 manifest | 不生成最终讲义 |
| `section-worker` | 为单小节写学习讲义草稿 | 不改全局索引 |
| `section-reviewer` | 审校单小节讲义 | 不直接越级发布 |
| `concept-synthesizer` | 基于已审校小节生成概念页候选 | 不凭空扩展概念 |
| `bridge-synthesizer` | 生成个人知识桥接页 | 不自动污染真实个人 Vault |
| `integrator` | 合并已通过门禁的产物 | 不发布未审校内容 |

### 10.3 可并行与不可并行

适合并行：

1. 不同小节草稿。
2. 不同小节审校。
3. 解析器试跑。
4. 从已审校小节提取概念候选。

不适合多 agent 同时直接写：

1. 首页。
2. 全书阅读地图。
3. 全局术语表。
4. 最终概念主文件。
5. 最终个人桥接总结。
6. QA 总报告。

## 11. Skill 设计

### 11.1 Skill 划分

建议首版至少有五个核心 skill：

| Skill | 作用 |
| --- | --- |
| `book-kb-orchestration` | 总流程、阶段门禁和文件契约 |
| `pdf-ingest-quality` | 来源解析、页码映射和提取风险 |
| `section-lesson-authoring` | 小节学习讲义撰写规则 |
| `section-lesson-review` | 忠实性、可学习性和风险审校 |
| `knowledge-bridge-synthesis` | 个人知识桥接页生成规则 |

### 11.2 Skill 编写原则

Skill 不应写成一个巨型提示词。首版建议把规则拆分为：

```text
SKILL.md
references/
templates/
schemas/
scripts/
```

其中：

- `SKILL.md` 只写触发条件、流程、输入输出、边界和必须加载的引用。
- 模板负责结构。
- schema 负责字段。
- review rubric 负责质量判断。
- 脚本负责确定性校验。

### 11.3 小节讲义 Skill 的关键约束

`section-lesson-authoring` 必须明确：

1. 目标是学习讲义，不是普通摘要。
2. 必须保留小节上下文。
3. 必须分轻重。
4. 必须写回原文条件。
5. 必须区分来源压缩、学习解释和个人桥接候选。
6. 对公式、图表、例题和 OCR 不确定处必须标风险。

## 12. Hook 与校验门禁

### 12.1 Hook 的定位

Hook 用于流程约束、写入边界、审计和 checkpoint。

Hook 不能替代：

- 内容理解。
- 学术正确性。
- 人类学习价值判断。
- 公式推导人工复核。

### 12.2 首版 Hook 建议

| Hook 目标 | 用途 |
| --- | --- |
| 写路径保护 | 限制 worker 只能写自己的 staging |
| Artifact 完整性 | worker 停止前检查产物是否齐全 |
| Schema 校验 | 缺字段、缺来源定位则失败 |
| 发布门禁 | 未 reviewed 内容不能进入 `study-kb` |
| Checkpoint | 任务中断时回写 manifest 状态 |
| 审计日志 | 记录 agent、section、阶段和结果 |

### 12.3 四道门禁

| 门禁 | 检查内容 |
| --- | --- |
| Ingest Gate | 来源定位、页码映射、解析质量和高风险类型 |
| Section Gate | 单节讲义字段、模板、来源和草稿结构 |
| Review Gate | 忠实性、可学习性、回原文提示和风险结论 |
| Publish Gate | 最终链接、索引、覆盖率、状态和阅读区纯净度 |

## 13. 文件契约

### 13.1 Section Worker 产物

```text
pipeline-workspace/staging/<section-id>/
├── source-slice.md
├── section-lesson-draft.md
├── section-metadata.yaml
└── worker-notes.md
```

### 13.2 Section Reviewer 产物

```text
pipeline-workspace/reviews/<section-id>/
├── review-report.md
├── review-decision.yaml
├── required-fixes.md              # 如需返工
└── fix-report-<topic>.md          # 可选：已 accept 后的小修审计
```

`fix-report-<topic>.md` 仅用于记录"已审校产物的小范围修复"，不能替代 `review-decision.yaml`。命名必须以 `fix-report-` 为前缀，不得随意起文件名。

### 13.3 发布产物

```text
study-kb/Section-Lessons/<section-id>-<title>.md
```

发布前至少检查：

1. 元数据完整。
2. 来源定位完整。
3. review decision 允许发布。
4. 高风险项已显式标记。
5. 文件命名、Frontmatter 和内部链接通过校验。

## 14. 社区参考与借鉴方向

当前可借鉴的社区方向可分为四类：

| 方向 | 代表参考 | 可借鉴点 |
| --- | --- | --- |
| Markdown 知识库编译 | OpenKB | 从原始资料编译 wiki、保留健康检查和结构化产物 |
| 书籍转 Agent 资料包 | book-to-skill | 按章节加载、术语表、速查表和书籍知识导出 |
| Wiki 维护规则 | llm-wiki 与相关 skill | 知识页 schema、审计、lint 和链接治理 |
| PDF 解析层 | Docling、Marker、PyMuPDF4LLM | Markdown、JSON、表格、图像和复杂 PDF 提取 |

这些参考可以指导设计，但首版框架仍需围绕“长篇资料的小节学习讲义”和“个人知识桥接”自建规则。

## 15. MVP 设计

### 15.1 MVP 范围

首版 MVP 不追求全书跑完，而是先验证学习效果和流程稳定性。

```text
1 份 PDF
3 类小节样本
1 个 manifest
1 套小节讲义模板
2 类核心 agent
3 道以上确定性校验
1 个可在 Obsidian 打开的 study-kb
```

### 15.2 样本选择

| 样本类型 | 验证问题 |
| --- | --- |
| 概念型小节 | 讲义是否真正易学 |
| 推导或公式型小节 | 风险标注和回原文机制是否可靠 |
| 强个人相关小节 | 知识桥接层是否有价值 |

### 15.3 MVP 产物

```text
study-kb/
├── Home.md
├── Learning-Maps/
│   ├── MOC-试点阅读地图.md
│   └── MOC-个人上下文优先路线.md
├── Section-Lessons/
│   ├── 3 篇小节学习讲义
├── Concept-Cards/
│   ├── 若干概念卡片
├── Knowledge-Bridges/
│   ├── 试点桥接页
├── Glossary/
│   ├── 试点术语表.md
└── Source-QA/
    ├── 来源页码索引.md
    └── 风险清单.md
```

### 15.4 MVP 验收标准

1. 读者阅读三篇小节讲义后，进入状态明显快于直接打开原 PDF。
2. 讲义保留原资料的学习主线，没有被原子化拆碎。
3. 推导、公式、图表和例题的高风险处能被显式标记，并能回到原文。
4. 个人知识桥接页能指出值得沉淀的知识、问题和输出线索，而不是泛泛关联。
5. 任务中断后可以按 manifest 恢复。
6. Obsidian 可以直接打开 `study-kb` 并浏览阅读地图、小节讲义和双链。

## 16. 首版排除项

首版明确不做：

1. 全书一键全自动发布。
2. 自动改写真实个人 Vault。
3. 复杂 Dataview 页面和插件依赖。
4. 向量数据库和通用 RAG。
5. 过度知识图谱分析。
6. 所有 PDF 类型的一次性兼容。
7. 首版就发布公共 plugin。

## 17. 主要风险与控制策略

| 风险 | 控制策略 |
| --- | --- |
| 小节切分错误 | 先建 manifest，再并行 |
| PDF 提取丢公式或图表 | Ingest Gate + 风险标记 + 回原文条件 |
| 多 agent 风格漂移 | 固定模板、skill、schema 和 review rubric |
| 多 agent 并发写冲突 | staging 隔离，integrator 单点合并 |
| AI 生成错误沉淀为知识 | reviewed 门禁，桥接建议优先 |
| 知识库重新变成长书 | A/B/C 轻重分级和阅读路线 |
| 过度碎片化 | Section-Lessons 为主阅读体 |
| 自动化覆盖用户理解 | 用户区块默认保护 |
| 版权与公开边界不清 | 开源引擎与模板，谨慎公开来源派生产物 |

## 18. 首个试点建议

对 `博弈论研究完全自学入门-自救白皮书.pdf` 的首个试点建议：

1. 先只选三个小节。
2. 三个小节分别代表概念型、推导型和个人上下文高相关型。
3. 先比较 PDF 解析质量，再确定首版解析路径。
4. 先把小节讲义模板和 reviewer rubric 调整到可读，再扩大并行规模。
5. 试点通过后，再决定是否扩展到整本资料、导出 book skill 或连接真实个人 Vault。

## 19. 结论

本框架的最终定位如下：

> 将长篇资料编译为一个以小节学习讲义为主阅读体、以 Markdown 为核心资产、兼容 Obsidian 阅读、可由多 Agent 分段生成和审校、可接入个人知识体系的本地学习知识库。

首版成功不等于“全书全自动生成完毕”，而等于：

1. 阅读体验比原文线性阅读更高效。
2. 来源、审校、风险和重跑机制足够稳。
3. 个人知识桥接层确实能帮助判断哪些内容值得长期沉淀。
4. 后续扩大到整本资料和其他长篇资料时，不需要推翻框架重来。
